⭐ Live Demo — Build a Skill in 5 Minutes

We'll build a Weekly Status Update skill — something everyone does.

1
Create the folder 
mkdir -p ~/.openclaw/workspace/skills/weekly-status-update
2
Write SKILL.md 
---
name: weekly-status-update
description: Generate a structured weekly status update from rough notes.
---

Take the rough weekly update notes and turn them into a structured status update.

## Required Sections
1. **Accomplishments This Week** (bullet points, max 5)
2. **Priorities for Next Week** (bullet points, max 5)
3. **Blockers & Risks** (anything that needs attention)
4. **Key Metrics / Numbers** (any quantifiable results)

## Rules
- Don't invent accomplishments not in the raw notes
- Keep the total output under 200 words
- Flag anything time-sensitive with ⚠️
3
Test it 
/new
"Here's what I did this week: [paste rough notes]"

That's the entire cycle. From zero to working skill in under 5 minutes.

  1. Record how you do something right now (screen recording, voice memo, or write down the steps)
  2. Transcribe what you recorded into plain language
  3. Feed it to AI and say: "Turn this into a SKILL.md for [platform]"
  4. Review and refine the output — fix gaps, add edge cases, test it
  5. Ship it and iterate based on how it performs
Why this works: You already have workflows you do manually. The skill is just capturing that expertise in a format an AI can follow. You're not inventing from scratch — you're documenting what you already know.

A general-purpose AI agent is the Chef — it knows how to cook. An AI Skill is the Recipe — a modular, reusable package of instructions. Without skills: the chef guesses. With skills: the chef executes precisely.

my-skill/
├── SKILL.md              (required — the recipe)
├── scripts/              (optional — executable code)
│   └── validate.sh
├── references/           (optional — docs loaded on demand)
│   └── api-docs.md
└── assets/               (optional — templates, images)
    └── template.md

SKILL.md has two parts: YAML Frontmatter (name, description) + Markdown Body (step-by-step instructions). A working skill can be as little as a 10-line markdown file.

Unlike recipes, skills are non-deterministic. The same skill may produce slightly different output each time. This is normal — treat skills as rough drafts that get better with iteration.

Three commands you need:

mkdir folder-name     # Create a new folder
cd folder-name         # Go into a folder
cat filename           # Read a file's contents

OpenClaw setup:

  • Sign up at campclaw.ai (free sandbox) or install locally at openclaw.ai
  • Skills live in ~/.openclaw/workspace/skills/<name>/SKILL.md
  • Install community skills: openclaw skills install <name>
  • Browse skills: clawhub.com
openclaw skills list             # See all loaded skills
openclaw skills install <name>    # Install from ClawHub
  • Skills should DO things, not TEACH things. Give focused execution steps.
  • Keep scope narrow. One skill, one job.
  • Test early with real inputs. Don't wait until it's "perfect."
  • The description field matters. That's how the agent discovers your skill.
  • First output won't be perfect. Iterate on the instructions, not the prompt.
Skill not triggering Description doesn't match use case → make it more specific
Output format wrong Instructions too vague → add explicit format examples
Agent ignores the skill Description doesn't cover use case → rewrite to match natural language

Cheat Sheet — Ctrl+P to print

🚀 Create a Skill (3 steps)

1. mkdir -p ~/.openclaw/workspace/skills/my-skill
2. Create SKILL.md inside that folder
3. /new then ask the agent to use it

📝 SKILL.md Template

---
name: my-skill
description: What it does & when to trigger it
---

# Instructions for the agent here
1. Step one
2. Step two
3. Output format

📂 OpenClaw Paths

Workspace:  ~/.openclaw/workspace/skills/<name>/
Shared:    ~/.openclaw/skills/<name>/
Install:   openclaw skills install <name>
List:      openclaw skills list

🔄 The Workflow

  1. Record something you do (screen/voice/steps)
  2. Feed to AI: "Turn this into a SKILL.md"
  3. Review & refine the output
  4. Test it, ship it, iterate

💡 Pro Tips

  • Keep SKILL.md under 500 lines
  • Put reference docs in references/
  • Put executable scripts in scripts/
  • Description field = how the agent finds it
  • Test early, test often
  • Skills can chain other skills
  • Use ClawHub to share with the community
🔒 Facilitator Notes & Advanced Topics ▾

Advanced Topics

Not covered in the training sprint — next steps for deeper exploration.

Skill Chaining

Skills can reference other skills in their instructions. Example: A weekly-report skill that chains the meeting-notes and github-issues skills to compose complex workflows from simple building blocks.

Multi-Agent Orchestration (OpenClaw)

  • Sub-agents: Spawn a separate agent to handle a task while the main agent continues
  • Task Flow: Durable multi-step workflow orchestration with state tracking
  • Lobster: Deterministic pipeline runtime with built-in approval gates

Progressive Disclosure with References

The references/ directory holds supplementary docs loaded only when needed, keeping SKILL.md concise while having deep domain knowledge available.

Publishing to ClawHub

clawhub sync --all        # Sync all workspace skills
clawhub publish <path>   # Publish a specific skill
